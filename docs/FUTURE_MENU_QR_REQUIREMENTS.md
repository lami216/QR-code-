# Future hosted menu and multi-link requirements

Phase 6 implements the menu tool honestly as a URL QR code to an existing menu. QR Studio does **not** host menu records. A future hosted menu would require a persistent restaurant/menu/item data model, authentication and ownership, publishing/versioning, media storage, allergen and accessibility fields, stable public URLs, moderation/security, privacy/retention rules, and an editing UX.

Likewise, the social tool supports one direct public profile URL. A hosted multi-link product would require an owned profile model, link ordering and validation, public page routing, abuse controls, account lifecycle, privacy choices, and optional analytics consent. It must not be represented as supported until those systems exist.
